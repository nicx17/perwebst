Mimick

Role: Engineering Student Repository: nicx17/mimick
Python Linux GTK4 Immich API

Overview

A daemon-based synchronization tool for intelligently uploading media files from a Linux desktop setup to an Immich server. Operating seamlessly as a background service, it watches local directories for new files and supports deep system integration including a system tray icon, notifications, and graphical settings management.

Features

  • File Monitoring: Utilizes inotify to immediately detect and queue new local files for upload.
  • Intelligent Connectivity: Automatically switches between Internal (LAN) and External (WAN) server URLs depending on availability using Captive Portal verification logic.
  • Desktop Integration: Complete with a system tray menu, About dialog, Status visibility, and a comprehensive settings UI for configuring sync folders, API keys, and server addresses.
  • Modern UI & Wayland Compatibility: Tested natively on GNOME environments utilizing Wayland, built with modern GTK4 standard libraries.
  • Concurrent Operations: Uses multi-threading (up to 10 workers) for fast batch-uploading.
  • Offline Reliability: Uploads that drop mid-flight or queue without a connection are serialized to a local cache and safely wait out offline statuses before resuming via background workers.
  • Automated Album Creation: Inherits your local folder names and creates corresponding albums seamlessly on the Immich server (e.g. ~/Pictures/Vacation routes directly to the "Vacation" album).
  • Secure Credential Storage: Safely stores the application API Key within the native system keyring (libsecret/KWallet).

Architecture & Operation

The solution is designed as a multi-threaded daemon with a safely decoupled UI process. Operations are split into specialized components ensuring rapid processing without sacrificing desktop stability.

  • File Monitor: Utilizes the watchdog library to hook into filesystem events (like IN_CREATE and IN_MOVED_TO). Crucially, it implements debouncing techniques to ensure files are fully written to disk before triggering, and immediately calculates SHA-1 checksums to support native Immich server-side deduplication.
  • Queue Manager: A thread-safe orchestrator receiving file tasks and distributing them across 10 daemon worker threads. Failed uploads are captured and persisted via a local JSON cache, standing ready to automatically resume upon network restoration even after a hard system reboot.
  • API Client: Encapsulates Immich server communication showcasing Intelligent Dual-URL support with failovers and captive portal verifications. Album creation uses strict thread-locking to prevent duplicate album records under simultaneous high-volume worker loads.
  • Decoupled UI: Communication between the daemon and graphical settings/tray app is handled efficiently via states tracked in cached JSONs limiting GTK/event-loop collisions, supplemented by notify-send for native OS desktop progress indicators.

System Setup

The app is tailored for Debian/Ubuntu, Fedora, and Arch-based systems and now ships as an easy to use AppImage. It supports Native Python environment installation and Systemd User Service persistence to automatically kickstart operations upon a user's local login.
