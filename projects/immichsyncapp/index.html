Immich Auto-Sync App

Role: Engineering Student Repository: nicx17/immich_sync_app
Python Linux Immich API Automation

Overview

A daemon-based synchronization tool for intelligently uploading media files from a Linux desktop setup to an Immich server. Operating seamlessly as a background service, it watches local directories for new files and supports deep system integration including a system tray icon, notifications, and graphical settings management.

Features

  • File Monitoring: Utilizes inotify to immediately detect and queue new local files for upload.
  • Intelligent Connectivity: Automatically switches between Internal (LAN) and External (WAN) server URLs depending on availability.
  • Desktop Integration: Complete with a system tray menu, About dialog, Status visibility, and a comprehensive settings UI for configuring sync folders, API keys, and server addresses.
  • Concurrent Operations: Uses multi-threading (up to 10 workers) for fast batch-uploading.
  • Automated Album Creation: Inherits your local folder names and creates corresponding albums seamlessly on the Immich server (e.g. ~/Pictures/Vacation routes directly to the "Vacation" album).
  • Secure Credential Storage: Safely stores the application API Key within the native system keyring (libsecret/KWallet).

Architecture & Operation

The solution comprises a backend daemon written in Python that oversees a synchronization queue populated by inotify events. A separated graphical tray component allows the user to interact with the daemon without needing the terminal, built via gobject-introspection and standard Linux AppIndicator libraries. Its additive-only approach ensures zero deletion or modification of source files securely.

System Setup

The app is tailored for Debian/Ubuntu, Fedora, and Arch-based systems. It supports Native Python environment installation and Systemd User Service persistence to automatically kickstart operations upon a user's local login.
