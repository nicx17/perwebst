Mimick

Role: Engineering Student Repository: nicx17/mimick
Rust Linux GTK4 Libadwaita Flatpak

Overview

Mimick is a Linux desktop sync client for Immich that watches local folders such as ~/Pictures and ~/Videos, uploads new media in the background, and gives the user a native GTK4 settings experience instead of a script-only workflow. It is designed as a one-way sync tool that stays out of the way day-to-day while still exposing the controls you need for album routing, startup behavior, and network targeting.

Key Features

  • Live File Monitoring: Watches selected folders for new media and waits for file size stability before queueing uploads.
  • Immich-Compatible Deduplication: Computes SHA-1 checksums before upload to match the same duplicate detection strategy used by the Immich mobile clients.
  • Fast Parallel Uploading: Streams files directly from disk across 10 worker tasks, keeping memory pressure low even during larger batches.
  • Retry Persistence: Failed uploads are written to ~/.cache/mimick/retries.json and replay automatically when the app is launched again.
  • Dual-URL Connectivity: Can switch between internal LAN and external WAN Immich endpoints based on what is reachable at the time.
  • Flexible Album Mapping: Lets each watched folder target an existing album, a custom album name, or an album derived from the local folder name.
  • Secure Secret Storage: Stores the Immich API key in the system keyring through libsecret instead of leaving credentials in plain text configuration.

Desktop Workflow

Mimick is a Rust application built around GTK4, Libadwaita, and a StatusNotifier tray flow for Linux desktops. The settings window handles watch-path selection, URL configuration, API key setup, and login autostart, while the background process keeps working even after the window is closed.

  • Startup Catch-Up: On launch, Mimick rescans watched folders and only queues media that is new, changed, or retargeted to a different album.
  • Clear Background Behavior: Closing the settings window hides the UI, while quitting from the tray or launcher fully stops the sync process.
  • Portal-Friendly Folder Access: Flatpak builds rely on selected-folder access instead of broad home-directory permissions, which fits modern Linux sandbox expectations better.
  • Autostart Support: Supports login startup through the desktop portal in Flatpak builds and through a native autostart entry outside Flatpak.

Installation & Usage

Mimick is currently distributed as a beta Linux application for modern Immich setups. The recommended install path is the project's Flatpak repository, which keeps updates straightforward across distributions.

flatpak remote-add --user --if-not-exists mimick-repo https://nicx17.github.io/mimick/mimick.flatpakrepo

flatpak install --user mimick-repo io.github.nicx17.mimick

For native builds, the repository also ships a standard Rust toolchain workflow using cargo build --release.
